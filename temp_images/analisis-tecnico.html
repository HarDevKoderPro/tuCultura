⚙️ 4. Opciones de Implementación

A
CSS Puro + Imagen Original Centrada ⭐ RECOMENDADA

Usar la imagen original (720×744) centrada con background-size: contain sobre un fondo CSS con gradiente radial que replique los colores del paisaje (cielo celeste → verde → turquesa). La esfera queda centrada naturalmente.

Código CSS propuesto:

.header {
  background-color: #4cbfa6;
  background-image: 
    url(../imagenes/cabecera-nueva.jpg);
  background-size: contain;
  background-position: center;
  background-repeat: no-repeat;
  /* Gradiente lateral que extiende los colores naturales */
  background-color: #5bb8a0; /* color promedio del borde de la imagen */
}
/* O mejor con gradiente: */
.header {
  background: 
    url(../imagenes/cabecera-nueva.jpg) center/contain no-repeat,
    linear-gradient(90deg, #3a8c5c 0%, #4cbfa6 20%, #6dd5d5 40%, 
                           #6dd5d5 60%, #4cbfa6 80%, #3a8c5c 100%);
}
  • ✅ Mínimos archivos a modificar (solo CSS)
  • ✅ Imagen no se deforma nunca
  • ✅ Peso ultra ligero (68KB original)
  • ✅ El gradiente CSS se adapta a cualquier ancho de pantalla
  • ❌ En desktop la imagen puede verse pequeña dentro del header de 180px
B
Imagen Expandida por IA + Responsive con Media Queries BUENA OPCIÓN

Usar las imágenes panorámicas ya generadas (3200×800 para desktop, 800×800 para mobile) con background-size: cover. Se sirven diferentes versiones según el viewport.

Código CSS propuesto:

/* Mobile-First */
.header {
  background-image: url(../imagenes/cabecera-mobile.jpg);
  background-size: cover;
  background-position: center;
}

/* Tablet */
@media (min-width: 768px) {
  .header {
    background-image: url(../imagenes/cabecera-tablet.jpg);
  }
}

/* Desktop */
@media (min-width: 900px) {
  .header {
    background-image: url(../imagenes/cabecera-desktop.jpg);
  }
}
  • ✅ Imagen ocupa 100% del espacio sin huecos
  • ✅ Esfera centrada en todas las resoluciones
  • ✅ Peso optimizado vs imagen actual (192KB vs 4.5MB)
  • ❌ Requiere generar/mantener múltiples archivos de imagen
  • ❌ Los bordes expandidos son gradientes (no paisaje real)
C
Híbrido: Imagen Expandida Desktop + CSS Gradient Fallback MÁS COMPLETA

Combina lo mejor de ambas: usa la imagen panorámica expandida como fondo principal, con un gradiente CSS como fallback y WebP para navegadores modernos. Incluye optimización con <picture> o CSS art-direction.

Código CSS propuesto:

/* Mobile-First */
.header {
  background: 
    url(../imagenes/cabecera-mobile.webp) center/cover no-repeat,
    url(../imagenes/cabecera-mobile.jpg) center/cover no-repeat,
    linear-gradient(180deg, #87ceeb 0%, #5ecce6 40%, #4cb848 100%);
}

@media (min-width: 900px) {
  .header {
    background: 
      url(../imagenes/cabecera-desktop.webp) center/cover no-repeat,
      url(../imagenes/cabecera-desktop.jpg) center/cover no-repeat,
      linear-gradient(90deg, #3a7c4c, #5ecce6, #3a7c4c);
  }
}
  • ✅ Máxima compatibilidad (JPG fallback + WebP optimizado)
  • ✅ Gradiente CSS si falla la carga de imagen
  • ✅ Excelente rendimiento (WebP: 76KB desktop, 52KB mobile)
  • ✅ Profesional en todos los dispositivos
  • ❌ Más complejidad en CSS

🎯 6. Mejoras de Estilo del Menú

Problema Actual

El texto del menú en desktop (color: #c4d6db) es gris-azulado claro. Sobre el nuevo fondo turquesa/celeste, el contraste será insuficiente.

Propuesta de Estilos para el Menú Desktop:

/* Texto con sombra para legibilidad sobre cualquier fondo */
header nav ul li a {
  color: #ffffff;
  font-weight: 600;
  text-shadow: 
    1px 1px 3px rgba(0, 0, 0, 0.7),
    0 0 8px rgba(0, 0, 0, 0.4);
  letter-spacing: 0.5px;
}

/* Fondo semitransparente para cada item */
header nav ul li {
  background: rgba(0, 0, 0, 0.25);
  border-radius: 6px;
  backdrop-filter: blur(4px);
}

header nav ul li:hover {
  background: rgba(0, 150, 136, 0.6);
  transform: translateY(-1px);
  transition: all 0.3s ease;
}
